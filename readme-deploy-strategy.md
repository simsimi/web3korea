# KOWA 홈페이지 Netlify 배포 전략

## 1. 배경

- **프로젝트**: 웹3기업협회(KOWA)의 공식 홈페이지
- **현황**: 단일 `index.html` 파일로 구성된 순수 정적 웹사이트. JavaScript를 이용해 동적인 UI/UX(애니메이션, 차트, 아코디언 등)를 구현함.
- **목표**: 안정적이고 관리가 용이하며, 향후 확장성을 고려한 호스팅 플랫폼에 웹사이트를 배포한다.
- **선택**: GitHub Pages도 좋은 대안이지만, CI/CD(지속적 통합/배포) 자동화, 손쉬운 커스텀 도메인 설정, 내장된 폼 처리 기능 등을 제공하는 Netlify를 최종 배포 플랫폼으로 선택한다.

## 2. 배포 계획 (Action Items)

### 단계 1: 소스코드 관리 (GitHub)
1.  **[완료]** 프로젝트를 위한 GitHub 리포지토리를 생성한다.
2.  **[완료]** `index.html` 파일을 리포지토리의 `main` 브랜치에 푸시한다.

### 단계 2: Netlify 배포
1.  **[예정]** Netlify에 GitHub 계정으로 가입 또는 로그인한다.
2.  **[예정]** 'Add new site' > 'Import an existing project'를 선택하여 GitHub 리포지토리와 연결한다.
3.  **[예정]** 배포할 KOWA 홈페이지 리포지토리를 선택한다.
4.  **[예정]** 빌드 설정을 구성한다.
    - **Build command**: 비워둠 (별도 빌드 과정 없음)
    - **Publish directory**: 비워둠 또는 `/` (리포지토리 루트)
5.  **[예정]** 'Deploy site' 버튼을 클릭하여 초기 배포를 진행하고, `*.netlify.app`으로 끝나는 임시 URL로 배포 결과를 확인한다.

### 단계 3: Netlify 폼(Form) 연동
1.  **[예정]** `index.html`의 `<form>` 태그에 `data-netlify="true"` 속성을 추가한다.
2.  **[예정]** 기존에 폼 제출을 가로채던 JavaScript 이벤트 리스너(`contactForm.addEventListener('submit', ...)` 부분)를 삭제 또는 주석 처리한다.
3.  **[예정]** 변경된 `index.html`을 GitHub 리포지토리에 푸시하여 Netlify 자동 배포를 트리거한다.
4.  **[예정]** 배포된 사이트에서 폼을 테스트하고, Netlify 대시보드의 'Forms' 탭에서 제출된 데이터가 정상적으로 수신되는지 확인한다.

### 단계 4: 커스텀 도메인 연결
1.  **[예정]** Netlify 사이트 대시보드의 'Domain management'로 이동한다.
2.  **[예정]** 구매한 커스텀 도메인(예: `www.kowa.or.kr`)을 추가한다.
3.  **[예정]** 도메인 구매처(가비아, 카페24 등)의 DNS 설정 페이지로 이동한다.
4.  **[예정]** Netlify에서 안내하는 DNS 레코드(보통 Apex 도메인에 대한 `A` 레코드, `www` 서브도메인에 대한 `CNAME` 레코드)를 설정한다.
5.  **[예정]** DNS 설정이 전파되고 Netlify가 이를 확인할 때까지 기다린다.
6.  **[예정]** Netlify에서 자동으로 발급/갱신해주는 SSL(HTTPS) 인증서가 적용되었는지 확인한다.

### 단계 5: 최종 검증
1.  **[예정]** 커스텀 도메인으로 사이트가 정상적으로 접속되는지 확인한다.
2.  **[예정]** 모든 페이지 내 링크와 기능이 올바르게 동작하는지 최종 테스트를 진행한다.
